interface SceneProcessor

package org.angle3d.post

implemented by FilterPostProcessor, PreDepthProcessor, AbstractShadowRenderer, BasicShadowRenderer

Scene processors are used to compute/render things before and after the classic render of the scene. They have to be added to a viewport and are rendered in the order they've been added

Methods

cleanup ():Void

Called when the SP is removed from the RM.

initialize (rm:RenderManager, vp:ViewPort):Void

Called in the render thread to initialize the scene processor.

Parameters:

rm

The render manager to which the SP was added to

vp

The viewport to which the SP is assigned

isInitialized ():Bool

Returns:

True if initialize() has been called on this SceneProcessor, false if otherwise.

postFrame (out:FrameBuffer):Void

Called after a frame has been rendered and the queue flushed.

Parameters:

out

The FB to which the scene was rendered.

postQueue (rq:RenderQueue):Void

Called after the scene graph has been queued, but before it is flushed.

Parameters:

rq

The render queue

preFrame (tpf:Float):Void

Called before a frame

Parameters:

tpf

Time per frame

reshape (vp:ViewPort, w:Int, h:Int):Void

Called when the resolution of the viewport has been changed.

Parameters:

null

vp
